npm, nodemon, git and cross-env

npm init or npm init -y

is used to initialize a new project, by setting up a npm package file.
The -y is for agreeing to every default configuration for a quick setup.

npm install or npm i

Is used to install already defined dependencies from the package.json file.
If followed by a library name, it will install the latest/default version automatically within the project. SUCH AS nodemon:

Nodemon is a tool we use while developing nodejs programs, as it help with refreshing the app, whenever a change is made.
When installing it, do so globally, as you might need it for more projects - and it makes no sense to have inside the project.

npm install -g nodemon

Or maybe even install it as a development dependency (not sure if development environment will be covered here)

npm install --save-dev nodemon


Git commands

The following command is used to stage our local changes. We use -A to send ALL changes.

git add -A

After that, you want to package the staged files prepare them to send it to the repository.

git commit -m "the dash m is for creating a msg, and this is the msg"

Now that the package is ready, you want to push it to the repository.

git push

And when you want to retrieve the repository, you will use pull

git pull

Git clone is a command for downloading existing source code from a remote repository. It is used like this:

git clone LINK-TO-REPOSITORY.GIT

We can use the git branch command for creating, listing and deleting branches.

git branch <branch-name>

This will create a local branch. If you want to push into a remote repository, use the following command:

git push -u <remote> <branch-name>

You can view branches with:

git branch or git branch --list

And delete one with

git branch -d <branch-name>

Git revert can be used to revert a commit make, but we need the hashcode of the commit. Therefore, we need to use this first:

git log --oneline

This will show the hashcode of a commit, and we can use that code to revert like this:

git revert HASHCODE

Whenever you've developed in a different branch and everything works, it might be time to merge it into the main development branch, or production.
First, make sure you're in the branch you want to merge into. Move into the dev branch:

git checkout dev

From here, you should first update the branch with pull or fetch. Then merge it:

git merge <branch-name>
